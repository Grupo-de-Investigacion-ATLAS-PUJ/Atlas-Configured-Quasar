LogIt

Logging in quasar is handled by the LogIt component. For instructions on how to use LogIt generally, i.e. how to log specific messages from a quasar server please refer to the LogIt repository.

Configuring LogIt in a Quasar Server

As described above, the general means of logging via LogIt is documented here. However, you will need to configure LogIt in your quasar server - this section describes the hooks etc. provided by quasar where you, the developer, are expected to insert configuration code.

Please read about LogIt components. It's not long or complicated and you need to know about them.

Presumably, the quasar server you write will perform tasks at runtime relating to different aspects. Examples may include accepting input (e.g. handling OPC-UA client item writes and method calls), sending commands to hardware (e.g. calling some 3rd party API) and handling disconnection/reconnection. The general idea of LogIt components is that each aspect has a dedicated logging component, the benefit of this is that LogIt allows you to raise and lower the logging verbositya> of components at runtime in order to tune your server's logging according to the aspect you are interested in.

Note: You, the quasar server developer, defines the components and decides whether log messages are considered TRACE, WARNING etc.

To Configure LogIt in a quasar server

The class QuasarServer is created in your project with a hook method for you to override with the LogIt component IDs specific to your server.
